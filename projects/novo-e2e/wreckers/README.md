This directory contains "wreckers" - these are patch files (generated by the "patch-package" NPM package) that apply some key, destructive change to NPM dependencies.

Use these to answer the question of "When our dependencies fail, do our test fail"? Ideally, the answer should be yes. Apply one of these wreckers, and then rerun tests on your component. If your tests cover enough functionality, then they should fail as a result. This grants us more confidence in our ability to automatically merge minor version updates to NPM dependencies, without needing to manually verify that they've left functionality alone.

To apply a patch, eg angular-imask.patch, run: `git apply --ignore-whitespace projects\novo-e2e\wreckers\angular-imask.patch`. At present, little of this process is automated.

For more information, look at the patch-package documentation:
https://www.npmjs.com/package/patch-package
