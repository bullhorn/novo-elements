🎉 April 2026 (version 13)
===========================

**Announcement**: Novo Elements v13 removes long-deprecated components and upgrades to ES-Module compatible dependencies.

Some key highlights include:
- Complete removal of the deprecated Ace Editor add-on.
- Upgrade to `novo-design-tokens` v0.1.x with new package export mappings.
- Removal of the `brace` dependency.
- GitHub Actions CI infrastructure modernized to Node.js 24 compatible versions.

## Upgrading to v13

- If your application is still using the Novo Elements `AceEditor` add-on, you **must** migrate to the [Code Editor](https://bullhorn.github.io/novo-elements/docs/#/utils/code%20editor) which uses @codemirror. The Ace Editor has been deprecated since v8 and is now fully removed.
- The `brace` package is no longer a dependency. If your application imported it through novo-elements, add it directly to your own `package.json`.
- Update your `novo-design-tokens` dependency to `^0.1.4`.

Notable changes [#](https://bullhorn.github.io/novo-elements/docs/#/updates/v13-announce#notable-changes){#notable-changes}

### Removed Features
- Ace Editor: Completely removed the deprecated Ace Editor add-on, module, and form control
- Removed the `brace` dependency from the project

### Condition Builder
- Allow ability to not recreate templates when calling reset on condition builder [#1771](https://github.com/bullhorn/novo-elements/pull/1771)
- Add clear condition function [#1771](https://github.com/bullhorn/novo-elements/pull/1771)
- Add `allowEmptyField` input [#1771](https://github.com/bullhorn/novo-elements/pull/1771)
- Check for field definition before setting default operator [#1771](https://github.com/bullhorn/novo-elements/pull/1771)

### Bug Fixes
- Memory: Change modalService and toastService to avoid maintaining ViewContainerRefs longer than needed [#1790](https://github.com/bullhorn/novo-elements/pull/1790)
- ToastService: Added `ownViewContainer` function for better lifecycle management
- Dependencies: Adjust `post-robot` dependency to be more permissive

### Code Quality
- Remove wildcard imports and add lint rules to prevent them [#1777](https://github.com/bullhorn/novo-elements/pull/1777)
- Add lint rule for extra semi-colons [#1777](https://github.com/bullhorn/novo-elements/pull/1777)

### Dependencies
- Upgraded `novo-design-tokens` from `^0.0.10` to `^0.1.4`

### Infrastructure
- Updated GitHub Actions (`checkout`, `setup-node`, `upload-artifact`, `download-artifact`) to Node.js 24 compatible versions
- Pinned npm version in CI workflows for consistent builds
